# Faithful Image Upscaler | 高清放大与 1:1 保真修复

## 目标

将用户提供的图片进行高清放大和清晰化处理。

核心原则：

> 保持原图 1:1 还原，只提升画质，不重新设计图片。

不要把图片重新生成成另一张更好看的图，而是把原图变得更清楚、更干净、更适合发布。

## 适用场景

适合处理：

- 表情包
- 头像
- 自媒体封面
- 海报
- UI 截图
- 产品图
- 老照片
- 模糊图片
- 低清素材
- 压缩严重的图片
- 带文字的图片

## 必须保留

处理时必须尽量保持以下内容不变：

1. 原图构图不变
2. 原图比例不变
3. 主体位置不变
4. 人物脸型、表情、姿势不变
5. 动物表情、动作不变
6. 文字内容不变
7. 字体风格尽量不变
8. 图标、贴纸、装饰元素不变
9. 背景风格不变
10. 色彩氛围不变
11. 原图整体风格不变

## 需要增强

只允许做以下增强：

1. 高清放大图片
2. 提升分辨率
3. 增强边缘锐度
4. 提升文字清晰度
5. 减少模糊
6. 减少压缩噪点
7. 减少马赛克感
8. 清理锯齿边缘
9. 优化细节质感
10. 让图片更适合自媒体、PPT、社交平台发布

## 禁止事项

不要做这些事：

- 不要重新设计图片
- 不要改变构图
- 不要改变主体姿势
- 不要改变人物身份特征
- 不要把真人变成插画
- 不要把插画变成真人
- 不要替换文字
- 不要改错文字
- 不要删除原图文字
- 不要添加新文字
- 不要添加新元素
- 不要改颜色风格
- 不要过度磨皮
- 不要把图片变得塑料感
- 不要把表情包修得太精致失去原味
- 不要改变原图的幽默感、情绪和视觉记忆点

## 处理策略

### 1. 普通照片

重点增强：

- 脸部清晰度
- 头发细节
- 皮肤自然质感
- 衣服纹理
- 光影层次
- 背景干净度

要求：

- 保持真人感
- 不要过度美颜
- 不要磨成 AI 皮肤
- 不要改变五官

### 2. 表情包

重点增强：

- 表情清晰
- 文字清晰
- 边缘干净
- 保留原本搞笑感

要求：

- 不要修得太高级
- 不要改变表情
- 不要改变文字内容
- 不要重新设计成新表情包

### 3. UI 截图

重点增强：

- 文字可读性
- 图标边缘
- 卡片边界
- 按钮形状
- 阴影层次

要求：

- 不要改 UI 布局
- 不要改组件位置
- 不要改文字内容
- 不要重新设计界面

### 4. 海报 / 封面

重点增强：

- 标题文字清晰
- 主视觉细节清晰
- 背景质感干净
- 装饰元素边缘锐利

要求：

- 不要改版式
- 不要替换人物
- 不要改变标题
- 不要重新设计封面

### 5. 长图 / 作品集 / 信息图

长图和文字密集图的高清标准不是“长边达到 4K”，而是文字、图标和小组件在查看时真的清楚。

重点增强：

- 小字号文字可读性
- 图标和插画边缘
- 卡片、分隔线、时间轴、按钮等 UI 元素边界
- 渐变、背景色块和阴影的干净度
- 整张长图上下各段的一致清晰度

要求：

- 不要只把长边放大到 3840px 就结束
- 不要把长图缩短、裁掉、压扁或改成普通海报比例
- 不要重排版面
- 不要改文字、标题、作品链接、联系方式或数字
- 对超长图优先按宽度或短边保证清晰度，再等比计算高度
- 必要时分段处理每个屏幕区块，增强后再按原始顺序无缝拼回，最终仍保持原图完整比例
- 优先使用真实像素增强、去噪、锐化、超分辨率和分段处理，不要用生成式重绘替代原图
- 如果原图本身是低清截图，小字已经无法辨认，不要凭空编造文字或生成看似更清楚但内容不同的新版本

## 文字密集图质量上限

对作品集、履历、海报、UI 长截图、信息图等文字密集图片，必须先判断源图是否包含足够信息。

如果原图里的小字已经糊成色块、笔画断裂、无法肉眼辨认，单纯高清放大无法恢复真实文字。此时应优先建议用户提供更接近源头的文件：

- 原始 PNG/JPG 导出图
- PDF
- Figma、Sketch、PSD、AI、PPT 等设计源文件
- 原始网页、HTML、截图前的页面链接
- 更高分辨率截图或未压缩图片

如果用户没有源文件，只能从低清图继续处理：

1. 明确说明“只能增强可见信息，不能保证恢复不可辨认文字”。
2. 继续做保真像素增强，不要改内容。
3. 如果用户接受重建文字，必须先 OCR/人工识别文字，再重排版复刻；这属于“重建/复刻”，不是 1:1 保真高清放大。
4. OCR 不确定的文字必须标注不确定，不要猜测联系方式、链接、日期、数字、姓名或标题。

## 默认输出要求

默认输出：

- 保持原图比例
- 输出高清 PNG
- 尽量放大到原图 2x 至 4x
- 保留原图视觉内容
- 输出结果应比原图更清晰、更干净、更适合发布

## 尺寸与比例规则

在处理前必须先读取原图宽高，并把原始尺寸写入判断过程。输出尺寸必须由原图尺寸按同一个缩放系数等比计算。

绝对禁止：

- 不要横竖互换
- 不要把竖图改成横图
- 不要把横图改成竖图
- 不要裁剪画面
- 不要添加白边、黑边或透明边
- 不要拉伸图片
- 不要压扁图片
- 不要强行改成 1:1、4:3、3:4、16:9 或其他平台比例
- 不要因为 4K 要求而把非 16:9 图片改成 3840x2160

尺寸计算规则：

1. 如果用户要求 2x、3x、4x，输出宽高分别为原图宽高乘以对应倍数。
2. 如果用户只说 4K 或高清 4K，先判断图片比例和内容：
   - 普通照片、头像、表情包等非长图：默认让原图长边放大到 3840px，短边按原比例计算。
   - 长图、作品集、信息图、UI 长截图、文字密集海报：不要按长边 3840px 作为目标；默认至少放大 2x，优先让短边或宽度达到足以阅读文字的尺寸，通常宽度不低于 2160px，必要时使用 3x 或 4x。
3. 如果用户要求短边 4K，才让短边放大到 3840px，长边按原比例计算。
4. 如果用户指定具体宽高，只有在该宽高与原图比例一致时才直接使用；比例不一致时，应说明会导致变形，并优先保持原比例。
5. 输出前必须核对：`输出宽 / 输出高` 应与 `原图宽 / 原图高` 基本一致。

长图判定规则：

- `高度 / 宽度 >= 1.8` 的竖图，通常视为长图。
- `宽度 / 高度 >= 1.8` 的横图，通常视为横向长图。
- 即使比例未达到 1.8，只要包含大量小字、UI 卡片、履历、作品集、时间线、信息图，也按文字密集图处理。
- 对长图，不要用“长边 3840px”作为默认高清目标，因为这会让短边过小，导致文字仍然模糊。

示例：

- 原图 `1239x1332`，要求 4x：输出 `4956x5328`
- 原图 `1239x1332`，要求长边 4K：输出约 `3572x3840`
- 原图 `1239x1332`，不能输出横向画幅，也不能输出 `3840x2160`
- 原图 `750x5000` 长图，只说高清 4K：不要输出 `576x3840`；应优先输出至少 `1500x10000`，更适合文字阅读时可用 `2250x15000` 或 `3000x20000`
- 原图 `1200x6000` 作品集长图，只说高清：优先 2x 或 3x，即 `2400x12000` 或 `3600x18000`，而不是把高度限制到 3840px

## 清晰度验收规则

输出前必须检查清晰度，而不只是检查像素尺寸。

验收重点：

- 小字边缘是否比原图更清楚
- 中文、英文、数字、联系方式、链接是否仍可辨认
- 图标、按钮、卡片边框是否更锐利
- 是否出现新文字、错字、乱码或文字糊成色块
- 长图上、中、下各区域清晰度是否一致
- 是否因为整图缩放导致长图文字仍然不可读
- 输出内容是否与原图一致，而不是换成了新的文案、联系方式、日期、头像、图标或版式
- 局部放大 100% 查看时，小字是否真的比原图更可读，而不只是整体看起来更鲜艳

如果输出尺寸变大但文字仍不清楚，应继续提高倍率、改用短边/宽度目标，或分段增强后再拼接。

如果经过 2x、3x 或 4x 保真增强后文字仍不可读，应停止继续生成同类结果，并向用户说明源图清晰度不足，建议提供原始导出图、PDF 或设计源文件。

失败示例：

- 输出尺寸更大，但小字仍然看不清。
- 输出看起来更干净，但原图文字被替换、改写或编造。
- 原图是作品集长图，生成结果变成另一份内容相似但文字、联系方式、时间线不同的新作品集。
- 原图的文字本来不可读，输出却出现了模型猜出来的新文字。

## 用户常用指令

当用户说：

```text
高清放大这张图
```

应理解为：

```text
保持原图内容、构图、比例、文字、主体和风格不变，对图片进行高清放大、细节锐化、噪点清理、压缩痕迹修复和文字清晰度增强，输出更清楚的高清版本。
```

当用户说：

```text
高清放大并 1:1 还原
```

应理解为：

```text
严格保留原图所有内容，不重新设计、不改构图、不改文字、不改主体，只做画质修复和清晰度增强。
```
