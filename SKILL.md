# Universal Faithful Image Enhancer | 通用图片高清处理

## 核心目标

对用户提供的任意图片进行高清放大、清晰化、细节增强和压缩痕迹修复。

核心原则：

> 保持原图内容、构图、比例、文字、人物、主体和风格不变，只提升清晰度和发布质量。

本 Skill 不是重新设计图片，不是重绘新图，不是替换内容，而是让原图更清楚、更干净、更适合发布。

## 强制处理流程

每次处理图片前，必须按以下流程执行。

### Step 1：读取原图信息

必须先判断：

- 原图宽度
- 原图高度
- 原图比例
- 是否为长图
- 是否为大图
- 是否为文字密集图
- 是否包含 UI、海报、简历、作品集、信息图、表格、小字号文字

判断规则：

- `高度 / 宽度 >= 1.8`：判定为竖向长图
- `宽度 / 高度 >= 1.8`：判定为横向长图
- 即使比例未达到 1.8，只要包含大量小字、卡片、UI、简历、作品集、时间线、表格，也按文字密集图处理
- 宽度小于 1200px 且包含大量文字时，判定为低宽度文字密集图
- 图片超过单次稳定处理尺寸时，必须分块或分段处理

### Step 2：选择高清策略

根据图片类型选择不同策略。

#### A. 普通照片 / 头像 / 产品图

使用整图高清增强。

处理重点：

- 提升分辨率
- 增强主体边缘
- 降低噪点
- 清理压缩痕迹
- 保持五官、表情、姿势、颜色不变

默认输出：

- 2x 或 4x
- 保持原比例
- 输出 PNG

#### B. 海报 / 封面 / 插画图

使用整图高清增强，必要时局部增强标题和主体边缘。

处理重点：

- 标题更清楚
- 主视觉更锐利
- 背景更干净
- 装饰元素边缘更清晰

禁止：

- 不要改标题
- 不要改版式
- 不要替换人物
- 不要新增元素
- 不要改变配色风格

默认输出：

- 2x 至 4x
- 保持原比例
- 输出 PNG

#### C. UI 截图 / 网页截图 / App 截图

使用文字保护型高清增强。

处理重点：

- 小字号文字可读性
- 图标边缘
- 卡片边界
- 按钮轮廓
- 分割线
- 阴影层次

禁止：

- 不要重排 UI
- 不要改组件位置
- 不要改文字
- 不要改按钮内容
- 不要把 UI 重新设计成另一版

默认输出：

- 至少 2x
- 文字密集时 3x 或 4x
- 保持原比例
- 输出 PNG

#### D. 长图 / 作品集 / 简历 / 信息图 / 文字密集海报

必须使用“分段高清 + 无缝拼接”策略。不要整图一次性模糊放大。

必须执行：

1. 按视觉模块切分图片，例如首屏、经历区、能力区、作品区、底部联系区。
2. 每段单独高清增强，重点增强小字号文字、数字、邮箱、链接、图标、卡片边框、时间线、按钮和插画边缘。
3. 每段保持原始比例和内容，不得重排，不得裁剪，不得改字。
4. 增强后按原顺序无缝拼回，拼接处不能出现断层、错位、白边、黑边、重复内容或缺失内容。
5. 输出完整长图，最终仍然是一张完整图片，不是多张分段图，除非用户明确要求分段输出。

## 长图尺寸规则

长图和文字密集图，不要默认按“长边 3840px”处理。长图如果只让长边到 3840px，宽度可能仍然很小，小字依旧看不清。

正确规则：

- 普通图：可以让长边到 3840px
- 长图 / 作品集 / 信息图 / UI 长截图：优先保证宽度，而不是长边
- 长图默认至少 2x
- 文字密集长图优先 3x
- 小字特别多时使用 4x

推荐宽度：

- 最低宽度：2160px
- 推荐宽度：2400px 至 3200px
- 文字特别密集：3200px 以上
- 原图本身宽度已经很大时，按 2x 处理即可

示例：

- 原图 `818x2048`：2x 为 `1636x4096`，3x 为 `2454x6144`，4x 为 `3272x8192`
- 作品集长图优先输出 3x 或 4x，而不是输出 `1535x3840`
- 原图 `750x5000`：不要输出 `576x3840`；推荐 `2250x15000` 或 `3000x20000`
- 原图 `1200x6000`：推荐 `2400x12000` 或 `3600x18000`，不要把高度限制到 3840

## 分段高清规则

当图片满足以下任意条件时，必须分段处理：

- 高度 / 宽度 >= 1.8
- 图片高度超过 3000px
- 图片包含大量小字号文字
- 图片是作品集长图、简历长图、网页长图、UI 长截图、信息图
- 整图处理后文字仍然模糊
- 整图处理会导致细节被平均化

分段方式：

- 优先按内容模块切分，而不是机械平均切分
- 推荐切分为首屏 / Header、中间内容模块、底部 / Footer
- 如果无法识别模块，可以按高度等分，但必须保留 50 至 100px 重叠区域，避免拼接断层

每一段都必须分别增强：

- 文字边缘
- 数字边缘
- 图标边缘
- 卡片边框
- 分割线
- 按钮轮廓
- 插画边界
- 压缩噪点
- 色块渐变断层

## 局部内容增强规则

长图分段后，不能只对每一段做整段等比放大。对于内容密集区域，必须做局部 ROI 二次增强。

ROI 指需要单独处理的局部区域，包括：

- 文字卡片
- 小字号段落
- 数字、日期、百分比
- 邮箱、电话、链接
- 图标、Logo、按钮
- 插画主体
- 产品截图、UI 组件
- 时间线、表格、标签和徽章

必须执行：

1. 在每个分段中识别关键 ROI。
2. 将 ROI 按原坐标裁出，单独进行更强的去噪、超分、锐化、对比度和文字边缘增强。
3. ROI 增强后必须贴回原坐标，不得改变大小、位置、内容和布局。
4. ROI 边缘必须自然融合，不能出现矩形边框、色差、重影或贴片感。
5. 最终输出前必须在 100% 缩放下检查 ROI，而不是只看整张缩略图。

对于用户截图中用红框标出的区域，必须把红框区域视为优先 ROI。红框内的文字、图标和插画如果仍然模糊，说明结果不合格，不能交付。

局部增强优先级：

1. 文字和数字
2. 联系方式、链接、日期、百分比
3. 卡片边框、按钮和分割线
4. 图标、Logo、插画主体
5. 背景渐变和装饰元素

如果 ROI 处理后仍然不清楚，应继续提高 ROI 局部倍率，或提示源图局部信息不足。不要通过生成式重绘猜测文字。

## 文字保护规则

文字密集图处理时，清晰度优先级如下：

1. 文字内容不能改
2. 数字不能改
3. 邮箱不能改
4. 链接不能改
5. 标题不能改
6. 按钮文案不能改
7. 文字位置不能变
8. 字体风格尽量保持
9. 只增强文字边缘和对比度

禁止：

- 不要猜测模糊文字
- 不要把文字重写成新内容
- 不要把乱码强行修成看似合理的文字
- 不要出现错字
- 不要新增不存在的文字
- 不要删除原本的文字

如果源图文字已经完全糊成色块，必须提示：

> 源图文字信息已经不足，无法保证 1:1 恢复文字内容。可以继续做画质增强，但如果要文字真正清楚，建议使用 SVG / HTML / Figma 矢量重建。

## 允许优化

只允许做以下优化：

- 等比放大
- 提升分辨率
- 增强锐度
- 提升文字边缘清晰度
- 降低噪点
- 清理压缩痕迹
- 减少锯齿
- 减少马赛克感
- 优化边缘细节
- 优化渐变色块平滑度
- 让图片更适合发布、展示、PPT、自媒体和网页使用

## 绝对禁止

无论处理什么图片，都禁止：

- 不要改变原图比例
- 不要裁剪画面
- 不要添加白边、黑边、透明边
- 不要横竖互换
- 不要把竖图改横图
- 不要把横图改竖图
- 不要强行改成 1:1、4:3、3:4、16:9
- 不要重排版面
- 不要替换文字
- 不要修改数字
- 不要修改联系方式
- 不要修改链接
- 不要新增元素
- 不要删除元素
- 不要换人物
- 不要换表情
- 不要改变动作
- 不要改变图标含义
- 不要改变整体风格
- 不要把图片重新设计成另一张图

## 输出前验收规则

输出前必须检查。

尺寸验收：

- 是否保持原图比例
- 是否按统一缩放系数放大
- 是否没有拉伸
- 是否没有压扁
- 是否没有裁剪
- 是否没有多余边框

内容验收：

- 原图主体是否保留
- 原图文字是否保留
- 原图数字是否保留
- 原图链接和联系方式是否保留
- 原图布局是否保留
- 原图风格是否保留

清晰度验收：

- 小字边缘是否比原图更清楚
- 中文、英文、数字是否可辨认
- 图标是否更锐利
- 卡片边框是否更清晰
- 按钮轮廓是否更清晰
- 长图上、中、下区域清晰度是否一致
- 是否没有出现新的错字、乱码、伪文字
- 红框、用户标注区域或明显重点区域是否经过局部 ROI 二次增强
- 100% 缩放查看重点卡片时，卡片内正文、数字、图标和插画是否真的比原图清楚

长图验收：

- 是否完整保留顶部
- 是否完整保留底部
- 是否没有中间缺失
- 是否没有拼接错位
- 是否没有重复段落
- 是否没有断层
- 是否没有局部清晰、局部模糊的问题

如果验收不通过，不能直接交付，必须重新处理。

## 返工规则

### 问题：图片变大了，但文字仍然糊

原因：

- 只做了整图放大
- 没有分段增强
- 宽度仍然不够
- 源图文字已经过小

修复：

- 改用 3x 或 4x
- 优先保证宽度达到 2400px 至 3200px
- 按模块分段增强后再拼接
- 对文字卡片、图标、插画和用户标注区域做 ROI 局部二次增强
- 在 100% 缩放下复查 ROI；如果 ROI 仍然糊，不能交付

### 问题：长图被压缩成普通比例

原因：

- 错误使用长边 3840px
- 没有保持原始比例

修复：

- 重新读取原图宽高
- 按统一缩放系数重新输出
- 不允许裁剪和改比例

### 问题：文字被改错

原因：

- AI 重绘或猜测文字

修复：

- 禁止重写文字
- 改为文字保护型增强
- 只增强边缘，不改内容
- 如果无法恢复，提示需要矢量重建

### 问题：拼接处断层

原因：

- 分段没有重叠区域
- 拼接顺序或位置错误

修复：

- 重新按模块切分
- 保留 50 至 100px 重叠区域
- 对齐后无缝拼接

## 用户短指令映射

当用户说：

```text
高清这张图
```

应理解为：

```text
读取原图尺寸和类型，保持原图内容、构图、比例、文字、主体和风格不变，选择适合图片类型的高清策略，进行等比放大、清晰化、锐化、降噪和压缩痕迹修复，输出更清楚、更干净、更适合发布的 PNG。
```

当用户说：

```text
高清放大这张长图
```

应理解为：

```text
先识别长图和文字密集区域，优先保证宽度和小字可读性，按视觉模块分段增强后无缝拼接，最终输出一张完整长图。不得改比例、不得裁剪、不得改字、不得重排版。
```

当用户说：

```text
4K 修复
```

应理解为：

```text
普通图可按长边 3840px 等比处理；长图和文字密集图不能只按长边 3840px，应优先 2x、3x 或 4x，并保证宽度足够让小字可读。
```

当用户说：

```text
1:1 还原高清
```

应理解为：

```text
严格保留原图所有内容、布局、比例、文字、数字、联系方式、人物和风格，只做画质增强，不进行重绘、重排、猜字或内容替换。
```
